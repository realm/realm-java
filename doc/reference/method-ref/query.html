Query Class

Query objects are used to build-up queries. The query classes are generated by the @Table annotation.

Query is build using a fluent interface (method invocations can be daisy chained). Conditions are by default folded using logical-and. Query objects can be reused, and queries can be repeatly executed by the find_all() method (resembles the prepare/execute pattern in SQL clients).

For String columns:

MethodDescription
Query equal/eq (String value)Represents 'equal to' criterium
Query notEqual/neq (String value)Represents 'not equal to' criterium
Query beginsWith (String value)Represents 'substring match at the begining of the text' criterium
Query endsWith (String value)Represents 'substring match at the end of the text' criterium
Query contains (String value)Represents 'substring search' criterium

Examples:

PeopleView v = people.hired.equals(true).age.between(20, 30).findAll();
